24.10.2020

Another point of the discussion was the embedding directives in the Templates.

Here I used some attribute pattern parsing $[path] for paths or $[(call)path key1="value1" key2="value2"... keyN="valueN"] for calls.

But one just have to integrate JSON in some arbitrary text. Since we use JSON I put some curly brackets around.

The end pattern is a dot outside the bracket. If the markup has content, it will set by a pipe.

To emphasize the directive I put a [=][=]=> before. Every whitespace before this pattern will be ignored, to make the template more readable.

A path value now will be set by =>{path}.

A JSON part will be embedded by ===>{JSON}. or ===>{JSON}|content=>{}., where content could be set for calls implementing ==>{GithubLinkCall->ModelConfig, CallContent}.

Here any combination of typed json could be set. If several calls are within, the content will set to the last call will set to

As a short version of the JSON with one call there is the possibility to use the pattern ==>{Call->value1, value2, ..., valueN}. and ==>{Call->value1, value2, ..., valueN}| ..content..=>{}. for calls with content.

This shorthand version is similar to the attribute version and simplify the usage for simple calls like the often used ==>{ConfigLinkCall->ModelConfig, CallContent}.:
