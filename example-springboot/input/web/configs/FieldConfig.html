⊚FieldConfig

The field configurations provide all necessary informations about the type of a field

Any $[(GithubLinkCall)EOR||FieldConfig.json configType="NONE" /] file from the classpath is loaded together with the ⊚model configurations into the configuration cache $[(GithubLinkCall)EO|models|EOConfigsCache.java configType="NONE" /].

In the spring boot example this is initialized by $[(GithubLinkCall)SP|web|EOConfigCached.java configType="NONE" /].

The separation of field and ⊚model configurations allow to avoid annoying, misleading and error prone definitions of fields for each model with the same name.

When the model is initialized via $[(GithubLinkCall)EOTTR||Models.json configType="NONE" /] class names, the fields will be added as [className].[fieldName].
