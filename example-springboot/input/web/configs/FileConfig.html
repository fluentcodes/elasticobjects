⊚FileConfig

The file configurations allows the access to predefined files on the server.

All Files named $[(GithubLinkCall)spring-resources||FileConfig.json configType="NONE" /] will be loaded from the classpath into the configuration cache $[(GithubLinkCall)eo|models|EOConfigsCache.java configType="NONE" /].

In the spring boot example this is initialized by $[(GithubLinkCall)spring|web|EOConfigCached.java configType="NONE" /].

Depending on the ConfigCacheKey different configuration classes with different methods will be stored.

  • $[(GithubLinkCall)FileConfig /]
  • $[(GithubLinkCall)ScsConfig /]
  • $[(GithubLinkCall)XlsxConfig /]

The configuration methods will be used by the different file calls.

  • $[(GithubLinkCall)FileReadCall /]
  • $[(GithubLinkCall)FileWriteCall /]
  • $[(GithubLinkCall)ScsReadCall /]
  • $[(GithubLinkCall)JsonReadCall /]
  • $[(GithubLinkCall)TemplateResourceCall /]
  • $[(GithubLinkCall)XlsxReadCall /]
