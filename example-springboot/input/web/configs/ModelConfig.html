⊚ModelConfig

The model configurations offer methods for accessing, setting or creating java objects via EO.

When the cache object $[(GithubLinkCall)eo|models|EOConfigsCache.java configType="NONE" /] is initialized any file in the classpath named $[(GithubLinkCall)eo-resources||ModelConfig.json configType="NONE" /] will be loaded.

In the spring boot example this is initialized by $[(GithubLinkCall)spring|web|EOConfigCached.java configType="NONE" /].

Each entry contains information about

  • ⊚Field names
  • Object access
  • Class generation
  • Database constraints

The cache then contains $[(GithubLinkCall)EOConfigMapModels /] model map with entries of the following types:

  • $[(GithubLinkCall)ModelConfigScalar /]
  • $[(GithubLinkCall)ModelConfigList /]
  • $[(GithubLinkCall)ModelConfigMap /]
  • $[(GithubLinkCall)ModelConfigObject /]

With superKey and interfaces fields it contains the necessary information about inheritance structure.
