⊚ModelConfig

The model configurations are for accessing, setting or creating objects via EO.

When the ≡cache is initialized any File in the classpath named ≡ModelConfig.json will be loaded.

Each entry contains information about

  • ⊚Field names
  • Object access
  • Class generation
  • Database constraints

The cache then contains ≡model map with entries of the following types:

  • ≡ModelConfigScalar
  • ≡ModelConfigList
  • ≡ModelConfigMap
  • ≡ModelConfigObject

With superKey and interfaces fields it contains the necessary information about inheritance structure.
