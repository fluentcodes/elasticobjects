⊚TemplateConfig

The template configurations allows the access to predefined templates on the server.

Templates are parsed by $[(GithubLinkCall)EO|calls.templates|Parser.java configType="NONE" /] for

  • $[<path>]
  • $[@<system variable>]
  • $[%<environment variable>]
  • $[(<call)<path <attributes>]
Special shortages for special calls are
  • TemplateCall: $[&<path <attributes>]
  • ValueCall: $[#<path <attributes>]

Templates allows a calling hierarchy returning a valuable content or just call things on the server. The json calls only allow a subsequent execution.

All Files named $[(GithubLinkCall)SPR||FileConfig.json configType="NONE" /] will be loaded. will be loaded from the classpath into the cache object $[(GithubLinkCall)EO|models|EOConfigsCache.java configType="NONE" /] the first time a template call will be initiated.

All pages in the examples are created by two templates:

  • $[(ConfigLinkCall)ContentPage.html configType="FileConfig" /]
  • $[(ConfigLinkCall)ConfigsPage.html configType="FileConfig" /]
This page is generated via $[(ConfigLinkCall)ContentPage.html configType="FileConfig" /]

On each page you will see a form with a text field containing a json call to create the same page via the generic post-call by eo-form $[(GithubLinkCall)SP|web|WebEo.java configType="NONE" /] and not with the get endpoints $[(GithubLinkCall)SP|web|WebEoGet.java configType="NONE" /] with url path variables.
