TemplateConfig

The template configurations allows the access to predefined templates on the server.

Templates are parsed by ≡Parser.java for

  • $[<path>]
  • $[@<system variable>]
  • $[%<environment variable>]
  • $[(<call)<path <attributes>]
Special shortages for special calls are
  • TemplateCall: $[&<path <attributes>]
  • ValueCall: $[#<path <attributes>]

Templates allows a calling hierarchy returning a valuable content or just call things on the server. The json calls only allow a subsequent execution.

All Files named ≡TemplateConfig.json will be loaded. will be loaded from the classpath into the ≡cache the first time a template call will be initiated.

All pages in the examples are created by two templates:

  • ⊚ContentPage.html
  • ⊚ConfigsPage.html
This page is generated via ⊚ContentPage.html

On each page you will see a form with a text field containing a json call to create the same page via the ≡generic post-call eo-form and not with the ≡get endpoints with url path variables.
