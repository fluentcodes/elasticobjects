EO - Elastic Objects

Elastic Objects is a generic wrapper for java native objects.

It's similar to Java 8 Optional but knows

  • the wrapper of the parent object,
  • all wrappers for the child objects by fieldName and
  • the type rules.

Combining these wrappers one can access, create or change java objects via path.

Since EO is generic, one can define calls with a simple generic method

  • execute(EO eo).

    • These calls could be combined in any order.

    With a JSON serialization/deserialization adding type to a fieldName, one can loosely transfer complex objects and trigger calls from a client.

    At a glance

    Java

    In a java application one could create a typed object with a simple path modifier (BasicTest) before the name. The BasicTest object then has the value "test" in testString.

    Synopsis

    A client is free to combine available values and calls. The only restriction is, that a configuration is defined on the server.

    On the server side one can add resources only by configurations. New Functionality can be implemented in pure Java and configurations.

    One generic interface offer all of the servers functionality to the client.

    Adding type and functional directives into JSON is only a small change with simple rules. This small change simplifies building and architecture of service oriented applications. But it enlarge application scope far beyond some disabled database operations.

    Type and Calls

    From a pattern perspective ElasticObjects implements a simple path pragma in Java, which determine type, structure and execution.

    It's JSON serializing mechanism allows to trigger the same remote e.g. via http.

    The template call allows to compose the calls in a hierarchical way.

    Spring Boot Demo

    This site itself is a demo for this principle: A small spring boot application together with Elastic Objects and some TemplateConfigurations.

    The page you see with an "ordinary" web-url triggers the spring boot method ≡WeEoGet.createRootPage with the MVC request mapping "/{selectedItem:.+}.html".

    This method creates at server side a new EO object with selectedItem="Home.html" and ≡TemplateResourceCall and the template name ≡ContentPage.html.

    When the ≡ContentPage.html template is parsed, it will load ≡Home.html. So this is what you read.
