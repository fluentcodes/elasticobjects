Entry Points

The spring boot configuration is minimal with only two classes defining the endpoints:

  • Generic post endpoints $[(GithubLinkCall)spring|web|WebEo.java configType="NONE" /] to generate EO template call on the client side
  • Get endpoints $[(GithubLinkCall)spring|web|WebEoGet.java configType="NONE" /] creating EO template call on the serverside by path variables.

Basically the demo is pure html, with dedicated urls for each page with $[(GithubLinkCall)spring|web|WebEo.java configType="NONE" /]. Only three main templates are actually defined:

  • $[(ConfigLinkCall)ContentPage.html configType="FileConfig" /]
  • $[(ConfigLinkCall)ExamplesPage.html configType="FileConfig" /]
  • <$[(ConfigLinkCall)ConfigsPage.html configType="FileConfig" /]

The layout origins from a wordpress theme with all its responsive magic. It's hidden in the $[(ConfigLinkCall)Header.html configType="FileConfig" /] and $[(ConfigLinkCall)Footer.html configType="FileConfig" /]. So the content page template $[(ConfigLinkCall)ContentPage.html configType="FileConfig" /] could be rather small.
