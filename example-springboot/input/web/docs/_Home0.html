Entry Points

The spring boot configuration is minimal with only two classes defining the endpoints:

  • Generic post endpoints ==>{GithubLinkCall->NONE, SP|web|WebEo.java}. to generate EO template call on the client side
  • Get endpoints ==>{GithubLinkCall->NONE, SP|web|WebEoGet.java}. creating EO template call on the serverside by path variables.

Basically the demo is pure html, with dedicated urls for each page with ==>{GithubLinkCall->NONE, SP|web|WebEo.java}.. Only three main templates are actually defined:

  • ==>{ConfigLinkCall->FileConfig, ContentPage.html}.
  • ==>{ConfigLinkCall->FileConfig, ExamplesPage.html}.
  • ==>{ConfigLinkCall->FileConfig, ConfigsPage.html}.

The layout origins from a wordpress theme with all its responsive magic. It's hidden in the ==>{ConfigLinkCall->FileConfig, Header.html}. and ==>{ConfigLinkCall->FileConfig, Footer.html}.. So the content page template ==>{ConfigLinkCall->FileConfig, ContentPage.html}. could be rather small.
