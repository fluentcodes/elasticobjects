Accessing Files

In the following example the content of \ ==>{GithubLinkCall->NONE, SPI|data.files|public.txt}. is set to the ==>{ConfigLinkCall->FieldConfig, targetPath}. xyz via ==>{GithubLinkCall->ModelConfig, FileReadCall}..

The configKey refers to a configuration in ==>{GithubLinkCall->NONE, SPR||FileConfig.json#L302}.

If you click, you will see the content is set to "xyz" since your role "guest" with the generic interface is in the read permissions.

If you remove "_serializationType" you can see the hidden fields used by EO, for example the duration of this call.
