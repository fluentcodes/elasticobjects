What is the Application Builder?

Application code is just formatted texts with certain rules. The application builder generate code by some structured data and templates.

This was the starting point for elastic objects some years ago. I created java code, tests, hibernate configuration or sql by simple excel sheet with some templates.

This help avoiding the boring part of programming by generating the repeating structures.

Part of ElasticObjects like $[(GithubLinkCall)AnObject /] are itself generated by templates and $[(GithubLinkCall)B|calls.generate|GenerateJavaConfigCall.java configType="NONE" /]. Here the generated sources serve as template itself keep control calls within with "keepCall". Outside the call directives one can edit code.

Actually there are three templates for generating java code:

  • $[(GithubLinkCall)Call /] by $[(GithubLinkCall)BR|templates.java.interface|INTERFACECreate.tpl configType="NONE" /]
  • $[(GithubLinkCall)CallImpl /] by $[(GithubLinkCall)BR|templates.java.bean|BEANCreate.tpl configType="NONE" /]
  • $[(GithubLinkCall)ConfigImpl /] by $[(GithubLinkCall)BR|templates.java.config|CONFIGCreate.tpl configType="NONE" /]

A control template combine loading data and generation call by $[(GithubLinkCall)BR|templates.java|JavaBuilder.tpl configType="NONE" /]

Furthermore the json configurations are generated by $[(GithubLinkCall)B|calls.generate|GenerateJsonCall.java configType="NONE" /]. The control template is $[(GithubLinkCall)BR|templates.json|JsonBuilder.tpl configType="NONE" /]
