What are the applications of EO

From a java code perspective on can use EO just as an object manipulation tool. The size of the core library with 270 KByte is rather small.

One can

  • serialize
  • deserialize
  • compare
  • convert
  • merge

simple java objects.

With the $[(GithubLinkCall)Call /] concept and the combination of calls one can create applications for almost any requirements.

There are two applications I use to check practicability:

  • ⊚ This web site elasticobjects.org by $[(GithubLinkCall)B||Builder.java configType="NONE" /]
  • ⊚ Code generation in module example-springboot with $[(GithubLinkCall)SP||Start.java configType="NONE" /]

These application takes heave use of the template calls:

  • $[(GithubLinkCall)TemplateCall /]
  • $[(GithubLinkCall)TemplateResourceCall /]
  • $[(GithubLinkCall)TemplateResourceStoreCall /]

Core of the template call is the $[(GithubLinkCall)EO|calls.templates|Parser.java configType="NONE"/] class.
