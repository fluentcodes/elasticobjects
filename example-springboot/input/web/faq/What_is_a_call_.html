What is a call?

A Call is just a special pojo extending CallImpl with a generic execute method with EO as input.

The attribute sourcePath defines the location of the input value. The attribute targetPath defines the output location.

There are two conditions to be set. The attribute condition is evaluated on the actual element. The attribute localCondition will be evaluated within sourcePath location.

If the attribute duration is set, when the call is executed. It's just informal. If set, it the call will not be executed.

There are no restriction, what could be implemented in an execution method. There are some really simple examples like the SinusValueCall.

The call works in a java application context when implemented. If a model configuration is defined, then the call can also be executed from a EO execute method.
