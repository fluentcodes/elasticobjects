What is a model configuration?

EO uses model configurations for each entry. They are provided by the EOConfigCache in the EORoot.

The EOConfigCache initially load every file named ModelConfig.json and Models.json from the class path.

The model configuration offers methods to

  • create
  • set
  • get
the subsequent object via field names. For java classes this is done via reflection invoking empty constructor, getter or setter.

The model configuration contains the rules for
  • allowed fieldNames
  • fieldValues via field configuration
.
There are five different ModelConfig classes implemented to load the entries in ModelConfig.json:
  • ModelConfigScalar
  • ModelConfigMap
  • ModelConfigList
  • ModelConfigObject
  • ModelConfigNone
