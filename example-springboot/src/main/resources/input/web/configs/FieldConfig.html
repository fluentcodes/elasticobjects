FieldConfig

The field configurations loaded in the cache together with ModelConfigs.html hold all necessary informations about the type of a field.

When the cache is initialized any File in the classpath named FieldConfig.json will be loaded.

The separation of field and model configurations allow to avoid annoying, misleading and error prone definitions of fields for each model with the same name.

When the model is initialize via Models.json class names the fields will be initialized via [modelName].[fieldName].
