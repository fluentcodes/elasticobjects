Actions are simple wrapper classes for a cached call configurations. Usually they extend the AssetCache to add rolePermissions field.

These caches could contain methods for access to a certain data source. The HibernateCache for example contains implicit session and wrapper methods for reading/writing.

Defined Actions

One can simply defines own actions. First implementation of some generic call are available.

  • JDBC/Hibernate access (DbSqlAction/HibernateModelAction)
  • Excel (XsltAction)
  • Csv (CsvAction)
  • Text File (FileAction)
  • Template (TemplateAction)

The call hierarchy

The call caches itself has a hierarchical structure.

This has the advantage that one has only define a database access once, even with several hibernate configurations.

In the start example actionKey has the value
HibernateModelAction.read(h2:mem:BasicTest).
Here an ActionBuilder create a new instance of the HibernateModelAction with the an instance var HibernateModelCache with the filterKey h2:mem:BasicTest.

HibernateModelCache

Defines the model, the permissons and the used HibernateCache. 

  "h2:mem:BasicTest": {
    "hibernateModelKey": "h2:mem:BasicTest",
    "hibernateKey": "h2:mem:basic",
    "description": "Basic H2 in Memory Test with BasicTest Table",
    "modelKey": "BasicTest",
    "scope": [
      "TEST"
    ]
    "rolePermissions": {
      "read": "guest",
      "write": "admin",
      "delete": "admin",
    },
  }

HibernateCache

Here the hibernate session factory is initialized. The connection to the DbCache defining the database schema is implicit via name cutting the last entry after “:” to “h2:mem”.

  "h2:mem:basic": {
    "hibernateKey": "h2:mem:basic",
    "description": "Basic H2 in Memory Test with config settings of the test resources.",
    "configFile": "test.config.hbm.xml",
    "auto": "create"
  }

DbCache

This cached value defines the schema and database type used. The connection to the HostCache defining access, port and protocol is implicit cutting the last entry after “:” to “h2” from dbKey value.

"h2:mem": {
    "dbKey": "h2:mem",
    "description": "Local in memory H2 using the test resources.",
    "extension": "DATABASE_TO_UPPER=false;",
    "schema": "mem:test",
    "driver": "org.h2.Driver",
    "jndi": "",
    "dbType": "H2"
  }

HostCache

This defines values accessing other server like port, protocol and user/password. 

  "h2": {
    "hostKey": "h2",
    "protocol": "jdbc:h2",
    "user": "sa",
    "rolePermissions": {},
    "module": "actions-basic",
    "subModule": "main",
    "expanded": false,
    "description": "H2 Host Definition",
    "naturalId": "h2"
  }
