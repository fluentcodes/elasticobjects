The model is the core for accessing object data via the Adapter. The model consists of two cached datatypes:

  • ModelCache
  • FieldCache

Each entry contains information about

  • Object access
  • Class Generation
  • Database Constraints

The example

In the example from the start page we create database request for persisted objects of the class BasicTest. 

  "BasicTest":  {
    "modelKey":"BasicTest",
    "table":"BasicTest",
    "hibernateAnnotations":false,
    "description":"Model class with different types of fields including generic collections, maps and {@link SubTest} objects.",
    "idKey":"id",
    "naturalKeys":["testString"],
    "fields": [
      "id",
      "basicTest",
      "subTest",
      "subTestList",
      "subTestMap",
      "untypedList",
      "untypedMap",
      "testInt",
      "testString",
      "testLong",
      "testDate",
      "testBoolean",
      "testFloat",
      "testDouble",
      "testObject"
    ],
    "shapeType":"BEAN",
    "module":"test-resources",
    "subModule":"main",
    "defaultImplementation":"",
    "superKey":"",
    "packagePath":"org.fluentcodes.projects.elasticobjects.assets",
    "author":"Werner Diwischek"
  }

The fields array contains keys corresponding with the FieldCache definitions e.g:

  "basicTest":  {
    "fieldKey":"basicTest",
    "fieldName":"basicTest",
    "description":"",
    "unique":"",
    "notNull":"",
    "length":"",
    "modelKeys":"BasicTest"
  }
