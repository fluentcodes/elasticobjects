The template call is a central part for creating source code and keep it up to date with the model configuration.

Beside overwriting the created target files each time the template process is started there is a special creation type defined, using the target itself as template, when the target file exists. The parsing the keep the template keys as comments in the target file.

This is useful when one want to add something to the target outside the template filterKey.

The template call is rather simple with

  • {insert…} for recursive calls
  • $[var] for variable settings.
